## Project Overview

Egg detector for a chicken coop. Pulls images from a Raspberry Pi via SSH/SCP, detects eggs using YOLO, and saves annotated results.

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Pull from Pi and detect
python detect.py

# Test with a local image
python detect.py --image path/to/test.jpg

# Debug mode (prints per-detection details)
python detect.py --image path/to/test.jpg --debug

# Prepare Label Studio exports for YOLO training
python prepare_dataset.py

# Train YOLO model
yolo train model=yolov8n.pt data=dataset/data.yaml epochs=50 imgsz=640
```

No automated test suite exists. Testing is done manually with local images via `--image`.

## Architecture

**Three-file Python project** (Python 3.12, no framework):

- **`detect.py`** — Main entry point with three phases:
  1. Image acquisition: SCP from Raspberry Pi (or local `--image` fallback)
  2. Detection: YOLO model (`detect_yolo`) — requires a trained model
  3. Output: Annotated image saved to `snapshots/annotated_*.jpg`

- **`detect_opencv.py`** — Standalone OpenCV heuristic detection (extracted from `detect.py`, not used in the main pipeline). Combines color-based HSV filtering (white/brown eggs) with Hough circle detection, then deduplicates overlapping detections by proximity.

- **`prepare_dataset.py`** — Converts Label Studio annotation exports into YOLO format with 80/20 train/val split, outputs to `dataset/`

## Key Configuration

- **`config.json`** — Runtime settings: Pi SSH credentials, model path (`models/best.pt`), snapshot directory. Gitignored; copy from `config.example.json`.
- **`config.yaml`** / **`dataset/data.yaml`** — YOLO dataset class definitions (auto-generated by `prepare_dataset.py`)

## Dependencies

opencv-python, numpy, ultralytics (YOLO), label-studio (annotation tool, not used at runtime)
